# Text2Pic 文字转长图生成器

Text2Pic 是一个完全基于前端（纯浏览器端执行）的轻量级长图排版生成工具。它可以将您输入的繁杂文本、段落以及插图，排版并渲染出带有优雅质感的高清长图。

## 核心特性

- **数据极度安全**：所有排版与渲染纯前端执行，不依赖任何后端数据库传输，保护隐私。
- **本地持久化**：采用 IndexedDB 持久化草稿与自定色彩，告别刷新即丢失。
- **闪电般渲染**：底层深度优化了字体缓存机制（html-to-image），摆脱无限加载，出图极速。
- **纯粹响应式界面**：内置 6:4 黄金分屏比例与实时全屏沉浸阅读。

## 技术栈

- **前端框架**：React 18 + TypeScript + Vite
- **样式方案**：Tailwind CSS (原生支持响应式与深浅色模式流体适配)
- **状态管理**：Zustand (极简的原子化状态器，实现全局配置的高效注入)
- **排版与渲染核心**：html-to-image (经过魔改并接入全局离线缓存，毫秒级静态渲染)
- **持久化基建**：localForage + IndexedDB (跨域海量 base64 图文与字体池存储)
- **组件及图标**：shadcn/ui 规范 + Lucide-React

## 功能概览

1. **模块化无缝混排**
   - 支持无限量添加纯文本段落与高清图片模块。
   - 所有段落和插图都自带流式布局能力，全自动适配右侧长图渲染器。

2. **图片支持与独立滑轨缩放**
   - 极速插入本地长图或配图。
   - 为每一张插入的图片提供独立的大小缩放滑轨 (10% - 100%)，支持即改即见。
   - 可在图片下方附加极简说明性说明 (Caption)。

3. **专业级极客排版**
   - **免费商用字体**：全域搭载 Noto Sans/Serif SC（思源黑体/宋体），导出长图直接商用无侵权困扰。
   - **Markdown 注入**：选中文本即可使用 `**文本**` 快捷实现文字的加粗高亮。
   - **BBCode 全彩染色**：内置色彩调色板，可保存您的专属色彩（`[color=#HEX]文本[/color]`），精准染色。
   - **撤销栈保护**：绝大多数格式化功能原生支持 `Ctrl+Z` 历史回退。

4. **动态深色模式**
   - 原生集成了深色模式与浅色模式。
   - 界面的深浅切换，导出的图片同样完美兼容对应的主题色阶渲染。

5. **云端缓存优化**
   - 针对庞大的中文字体启用永久 IndexDB 离线缓存池。第一秒打包完毕后，后续每次导出高清原图的时间将被大幅压缩至毫秒级。

## 部署与运行指南

由于 Text2Pic 是一个构建后的纯静态单页应用 (SPA)，它的部署成本极低。这里提供三种最主流的部署/运行方式：

### 方式一：Docker 构建部署（推荐）

推荐使用 Docker Compose 部署此服务，提供干净的进程隔离并在秒级启动。

1. **环境要求**：确保机器安装了 `Docker` 与 `Docker Compose`。
2. **启动命令**：在项目根目录执行：
   ```bash
   docker-compose up -d --build
   ```
3. **访问使用**：
   容器默认将内部 Nginx 服务的 80 端口映射到了主机的 `5173` 端口。
   在浏览器访问：`http://localhost:5173`

### 方式二：Node.js 本地运行

1. **环境要求**：需安装 Node.js (推荐 v18+)。
2. **下载与安装依赖**：
   ```bash
   npm install
   ```
3. **启动开发者服务器**：
   ```bash
   npm run dev
   ```
4. **访问使用**：
   在浏览器输入控制台输出的地址（通常为 `http://localhost:5173`）。

### 方式三：静态服务器托管

1. 编译静态产物：
   ```bash
   npm run build
   ```
2. 打包完成后，您会得到一个 `dist/` 文件夹。
3. 把 `dist/` 文件夹里的所有文件部署到传统的 Nginx, Apache, GitHub Pages 或 Vercel 等静态托管服务器。
*(注：若是 GitHub Pages 部署，请确保在 `vite.config.ts` 中配置了正确的 `base` 路径)*
